# 7人6副牌找朋友

这是一个 7 人、6 副牌的「找朋友」联机版项目。首版目标是本地可玩：账号登录、创建房间、AI 补位、服务端规则校验、完整一局、事件复盘和规则分析。

## 启动

```bash
npm install
npm run test
npm run dev
```

默认地址：

- Web：`http://localhost:5173`
- API/WebSocket：`http://localhost:8787`

## 已实现的核心房规

- 7 人、6 副牌、每人 45 张、底牌 9 张。
- 首轮庄家随机，之后按座位轮庄；庄家队下台时，下一轮跳过上一轮庄家队成员。
- 主级牌等于庄家自己的级。
- 亮主/反底需要任意两张王加同花级牌；无主需要 2 猫加同类小王或大王，级牌仍是常主但没有主花色级牌；反底不反庄。
- 庄家出第一手前叫两张「某花色第 N 张 A」找朋友。
- 暗友分数先按个人暂存，身份暴露后追溯进庄家队，未叫上的进闲家队。
- 扣底不能扣任何 A。
- 闲家末墩赢才抠底，倍数为 `2^末墩每家出牌张数`：单抠 2 倍，双抠 4 倍。
- 抓分：`0` 分大光庄家队升3级，`1-119` 小光庄家队升2级，`120-239` 庄家队升1级，`>=240` 闲家下台并升级。
- 闲家末墩抠底拿到的加分计入最终「闲家总抓分」。
- J/A 必打抠底：主J打到底7；副J按各人级数打回，8/9到7、10到8、J/Q到9、K到10、A到J；主A打到底J；副A打到K。庄家和已暴露朋友同属庄家队，按各自级数执行。
- 反底/扣底是「带9扣9」：原9张底牌先进手，再从手里扣回9张；扣回的牌可含原底牌，但不能含任何 A。
- 同门可甩，主牌也算一门；若甩牌中任一张/型能被某家压住，则甩牌失败并强制只出其中最小一手。

## 项目结构

- `packages/rules`：纯规则引擎、AI 策略和复盘分析。
- `apps/server`：账号、房间、SQLite 事件持久化和 WebSocket。
- `apps/web`：React 牌桌和复盘界面。
